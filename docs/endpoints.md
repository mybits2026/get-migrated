# Endpoints

This document defines the public routes and API contract for the application.

## HTTP Routes

| Route | Method | Type | Description |
| --- | --- | --- | --- |
| `/` | `GET` | Page | Home page with hero CTA and contact form. |
| `/api/contact` | `POST` | API | Accepts lead details and sends an email via Resend. |
| `/robots.txt` | `GET` | SEO | Crawling directives generated by App Router route handler. |
| `/sitemap.xml` | `GET` | SEO | Sitemap generated from app routes. |
| `/opengraph-image` | `GET` | SEO | Open Graph image endpoint. |
| `/twitter-image` | `GET` | SEO | Twitter card image endpoint. |

## API Contract: POST /api/contact

### Request

Headers:

- `Content-Type: application/json`

Body:

```json
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "message": "I need guidance for migration pathways."
}
```

Validation rules:

- `name`, `email`, `message` are required.
- `name` max length: `120`.
- `email` max length: `320` and must match basic email format.
- `message` max length: `4000`.

### Responses

`200 OK`

```json
{
  "ok": true,
  "message": "Lead submitted successfully."
}
```

`400 Bad Request`

```json
{
  "ok": false,
  "message": "Name, email, and message are required."
}
```

`500 Internal Server Error`

```json
{
  "ok": false,
  "message": "We could not submit your message right now. Please try again shortly."
}
```

## Operational Notes

- Endpoint runtime is `nodejs` to support the Resend SDK.
- Email sends fail when `RESEND_API_KEY` or `CONTACT_TO_EMAIL` are not configured.
- `CONTACT_FROM_EMAIL` defaults to `onboarding@resend.dev`.
